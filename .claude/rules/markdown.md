# Markdown Editing Rules

Applies to all Markdown files in the project

## Basic Rules

- Do not use Markdown bold notation (`**text**`)
- Do not use colons at the end of sentences (`sentence:`)
- Do not use emojis
- Do not wrap URLs with `<>` (use code notation or standard link format)
- Use clear and concise English

## markdownlint Rules

- MD040: Specify language for code blocks (`python`, `typescript`, `text`, etc.)
- MD036: Do not use bold as headings (use `###` for section separation)
- MD022: Blank lines before and after headings
- MD025: Do not use multiple H1 headings
- MD032: Blank lines before and after lists
