## 記載ルール（生成時の参考。出力には含めない）

### 基本ルール

- 発言の最後に必ず（発言者名）を付ける
- インデントで会話の流れを表現する
    - 最初の質問・意見・発言: インデントなし `-`
    - それに対する回答・返答: スペース4つ `    -`
    - さらなる追加質問・確認: スペース8つ `        -`
    - 新しい話題に移る場合はインデントなし `-` から開始
- 討議事項には議論・質疑応答・意見交換の内容のみを記載
- 一方的な説明や報告のみの内容は「その他・内部向けメモ」に記載

### 用語・表記

- 英語由来の専門用語・技術用語は英語表記する
    - 例: git merge, deploy, CI/CD, Terraform, backlog, sprint
- 一般的に定着したカタカナ語（プロジェクト、メンバー、スケジュール等）はカタカナのまま
- フィラーワード（「えーと」「あの」「なんか」「まあ」等）は除去する

### 要約の粒度

- 各発言は1-2文に要約する
- 数値・日付・金額・固有名詞は原文通り保持する
- 音声内容の抜け漏れがないよう、全ての議論を網羅的に記載

### 正確性

- 会議で発言されていない情報を追加しない
- 不明な場合は「{不明}」と記載する
- 発言者が特定できない場合は「（{推定名}※推定）」と注記する

### Todo の抽出

- 「～してください」「～をお願いします」「次回までに～」等はTodo候補として抽出する
- 担当者が不明の場合は「{未定}」とする
- 期限が明示されなかった場合は「期限: 未定」と記載する

### フィードバックの反映

- `references/feedback.md` が存在する場合は読み込み、過去のフィードバック内容を反映する
